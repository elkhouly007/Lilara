# Lilara audit — 2026-05-30

**Scope:** holistic, evidence-led audit of the Lilara runtime, hooks, adapters, fixtures,
eval/replay corpus, scripts, docs/ADRs, OWASP coverage map, and decision lattice. Baseline:
`master` @ 435883c, VERSION 0.1.6.

**Method:** a read-only multi-agent sweep across three defect-class clusters (engine gates /
scope-IO-availability / assurance-hygiene), then **per-finding verification against the live
engine** — every candidate was reproduced (or refuted) with a `node` probe of `decide()` or the
target function before being acted on. Verification refuted two agent-reported "critical"
findings and surfaced new ones, so only reproduced findings are carried here.

**A note on measurement discipline.** During execution I published an unverified magnitude for
the `run-fixtures.sh` leak (an early "392/run" figure) that came from a flawed measurement — I had
run a *copy* of the script from a temp path, which broke its `root="$(dirname "$0")/.."`
derivation and aborted it early. Re-measured **in place** across 3 serial trials, the real figure
is **~172 temp entries/run**. Every number in this document is the in-place measured one.

### Saw but did NOT act on (with reason)

- **base64/hex/URL-encoded MCP args** decode to dangerous commands but classify `generic` —
  intentional per ADR-008 (speculative decode = high FP). Folded into ADR-020 as the broader
  question.
- **Symbol-keyed MCP arg containers** escape `Object.values` extraction — not representable in
  JSON or any real MCP transport; negligible.
- **F25/F26 256 KB / 1000-node caps** — verified **fail-safe** (→ `unscannable` → require-review),
  not fail-open. Correct as-is.
- **F25/F26 outer `catch { fire:false }`** — looks fail-open, but the inner `_extractStringValues`
  already catches throws and returns `truncated:true` → `unscannable` → require-review, so the
  outer catch is near-unreachable. Posture nit, not a reachable bug.
- **glob-match anchors / state-paths `path.resolve` traversal / network-egress wildcard / IDN
  punycode / secret-scan min-length / markdown-link 64 KB cap / 6-adapter parity** — probed,
  verified safe.
- **`audit-local.sh` PID-keyed temp** (`$$`) vs `mktemp` — read-only audit tool, low risk; the
  `horus`→`lilara` rename of that line is in PR #85, but migrating it to `mktemp` was out of scope
  for a cleanup-only PR.
- **OWASP ASI05 "VERIFIED" wording** for Codex/Antegravity — the NOT COVERED section already
  discloses the live-E2E gap; a minor doc nit, not a coverage overstatement worth a PR.

### Refuted (agent-reported but disproved by verification)

- **"Critical: `r${m} -rf /` silent allow."** The reporting agent never ran a full `decide()`.
  Reproduced: `r${m} -rf /` on the **Bash** path → **block risk 10**. (`${m}` is an unset var →
  expands to `r -rf /`, not an attack; the real, narrower vuln is `${IFS}`-specific and MCP-only —
  shipped as PR #81.) The "all three `_classifyCommandDual` call sites identical / no parity gap"
  claim was correct.
- **"run-fixtures traps are correct."** One cluster asserted the script's isolation was clean; it
  is not (PR #83).

### Discovered during execution (not in the original sweep)

7. **Pre-existing red test masked by a newer floor (F24 shadowing).** `tests/runtime/
   change-intent.test.js`'s "F20 medium severity" test used fixture path `/proj/src/secret.js`,
   which **F24** (`credential-persistence-write`, rung 17.625, added in v0.1.6 *after* that test)
   hard-blocks before F20 (rung 18.5) is reached — so the decision is
   `block`/`credential-persistence-write-denied`, and the test had been **red on master**. Fixed
   in PR #84 (co-located, same file): fixture moved to a non-credential path. **Root cause it
   exposes:** `change-intent.test.js` is wired into **no CI gate** (`check-runtime-core.sh` runs a
   fixed *list* of `tests/runtime/*.test.js`, and this file is not on it), so its failure was
   invisible. See quality signal #4.

## Quality signals (for triage — surfaced, not all fixed)

1. **Eval gate was looser than the stated bar** (10/20 vs 0/0) — fixed in PR #86.
2. **`eval "$( … )"` legit-idiom blocking** (`ssh-agent`, `direnv`, `rbenv`, `pyenv` init) → block
   risk 10. An FP-risk trend the eval corpus does not exercise (no `eval "$(…)"` safe entries), so
   0/0 gives false comfort here. See ADR-019.
3. **MCP danger floor is narrower than the Bash path** beyond `${IFS}` (base64-pipe,
   network-process-sub, variable-as-command). See ADR-020.
4. **`tests/runtime/*.test.js` coverage is partial in CI.** `check-runtime-core.sh` runs an
   *explicit list* of runtime unit tests; files not on the list (e.g. `change-intent.test.js`,
   and `command-normalize.test.js` until PR-1 wired its check) can be red without failing CI. A
   glob-all-runtime-tests gate would catch finding 7's class. Not actioned (would surface unknown
   pre-existing reds — needs a triage pass first).
5. **`decision-engine.js` is ~2250 lines / monolithic** — maintainability signal; not actioned.
6. **`full-power-status.md` carries stale fixture counts** (386/386 at several lines while the
   latest entry is 390/405) — pre-existing doc drift; `check-fixture-count.sh` only requires *some*
   matching count to exist, so it passes. Worth a docs sweep.
