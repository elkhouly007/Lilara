## Deferred

- **Cryptographic agent identity / inter-agent trust protocol** (relevant to ASI09): Out of scope
  for a single-host Claude Code / OpenCode / OpenClaw helper. There is no multi-agent network
  to protect. Explicitly deferred — single-host scope.
  See: Microsoft Agent Governance Toolkit for a reference implementation.

## NOT COVERED

- NLP-level prompt injection detection (ASI01): Would require an LLM-in-the-loop classifier;
  out of scope for a local hook that must complete in <1 ms.
- Cryptographic inter-agent trust (ASI09): Deferred — single-host scope (see above).
- Codex and Antegravity PostToolUse API verification (ASI05): both adapters are source-trace verified (Codex 2026-05-24 via openai/codex; Antegravity 2026-05-24 via google-gemini/gemini-cli). Live end-to-end hook fires with a real `codex` or `agy` instance are still pending; documented as re-check triggers in their respective WIRING_PLAN.md files. The previous incorrect statement ("other harnesses lack PostToolUse") has been removed from ASI01/ASI04/post-adapter-factory.js; all 6 have wired adapters.
- Graduated protection for dual-use MCP args (P1, 2026-05-29): MCP tool calls carrying `destructive-db` / `auto-download` / `global-pkg-install` command strings now gate to require-review on the Bash-path-parity principle — but a trusted server (policy:allow) with a `DROP TABLE` arg is still allowed (that is the DB connector's legitimate data path). For a rug-pulled trusted server carrying a HARD_BLOCK class (rm -rf /, curl|sh, etc.) the bypass seam is closed: F25 now degrades to require-review rather than silent skip.
